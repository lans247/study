1. 문제 설명
이번 프로젝트에서는 힙을 이용한 정렬 프로그램을 구현할 것입니다. 구체적으로 최소 힙과 최대 힙을 유지하면서 입력 파일에서 명령어 목록을 읽어오는 프로그램을 작성하여 실행할 것입니다. 
중복되는 데이터는 힙에 삽입할 수 있습니다.
프로그램에서 지원해야 하는 명령어는 다음 세 가지입니다:
(1) 삽입하다
- 이 명령은 최소 힙과 최대 힙 모두에 정수를 삽입합니다. 행에 "INSERT 3"이라고 적혀 있으면 프로그램은 데이터 = 3인 새 요소를 최소 힙과 최대 힙에 삽입해야 합니다.

(2) 오름차순으로 데이터 인쇄
- "ASCEND" 명령은 힙의 요소를 오름차순으로 출력합니다. 이 때
명령어를 실행하려면 프로그램이 "hw2_result.txt"라는 이름의 출력 파일에 정렬된 요소를 포함하는 행을 작성해야 합니다. (지정된 이진 파일을 실행해 보십시오.) 
- 이 명령어를 실행하려면 min 힙을 사용해야 합니다. 데이터를 오름차순으로 인쇄한 후에도 모든 요소가 포함된 min 힙이 있어야 합니다. 
이 작업을 수행하려면 min 힙에서 요소를 삭제해야 하므로 요소를 삭제하기 전에 min 힙의 복사본을 만들어야 합니다.

(3) 내림차순으로 데이터 인쇄
- "DESCend" 명령어는 힙의 요소를 내림차순으로 출력합니다. 이 명령어에서 프로그램은 "hw2_result.txt"라는 이름의 출력 파일에 정렬된 요소를 포함하는 행을 작성해야 합니다. (지정된 이진 파일을 실행해 보십시오.)
- 이 명령을 실행하려면 max heap을 사용해야 합니다. 데이터를 내림차순으로 인쇄한 후에도 max heap에 모든 요소가 포함되어 있어야 합니다. 
이 작업을 수행하려면 min heap에서 요소를 삭제해야 하므로 요소를 삭제하기 전에 max heap의 복사본을 만들어야 합니다.

2. 요구사항(세심하게 읽어보세요!)
(1) 당신은 hw2라는 이름의 단일 C 프로그램을 작성해야 합니다.

(2) 프로그램은 입력 파일 이름인 명령줄 인수 하나를 사용합니다. 예를 들어, 사용자는 다음과 같이 프로그램을 실행합니다. ./hw2 input.txt
첫 번째 인수는 입력 파일의 이름입니다.

(3) 명령줄 인수의 수가 1이 아니면(0 또는 1보다 큼) 사용 문자열을 디스플레이에 인쇄하고 프로그램을 종료해야 합니다. 사용 문자열은 다음과 같습니다:
용도 : ./hw2 입력_filename

(4) 입력 파일이 존재하지 않으면 프로그램은 화면에 오류 메시지를 표시하고 종료해야 합니다. 오류 메시지는 다음과 같습니다. 입력 파일이 존재하지 않습니다.

(5) 입력 파일 형식은 다음과 같습니다. 각 행에는 하나의 명령어가 들어 있습니다.
인서트 5
인서트 8
인서트 3
상승
인서트 10
내림차순
참고용으로 예시적인 입력 파일이 제공될 것입니다. 평가에서 TA는 다른 입력 파일을 사용할 것입니다. 
입력 파일의 형식이 항상 정확하다고 가정할 수 있습니다. 
입력 파일의 내용에 대해 오류 처리를 할 필요가 없습니다.
(6) 프로그램이 끝나면 화면에 다음 행을 인쇄해야 합니다. 
output written to hw2_result.txt
running time : 0.052411 seconds
실제 실행 시간은 실행 시마다 다를 것입니다. 
실행 시간을 출력하려면 clock() 를 사용합니다.

(7) 채점조교는 make를 사용합니다. 실행 이진파일 hw2를 생성하십시오. 

%min, max heap은 1,000,000개의 element를 가집니다. 

3. 제출
소스 코드와 Makefile을 제출해야 합니다. (컴파일된 바이너리 파일은 제출할 필요가 없습니다.) 파일을 cse3080_hw2_20200001.zip 파일로 결합합니다. 빨간색 숫자는 학생 ID로 변경해야 합니다. 사이버 캠퍼스에 파일을 제출하세요.
4. 평가기준
(1) 당신의 프로그램은 바이너리 파일을 생성하기 위해 잘 컴파일되어야 합니다.
(2) 사용자가 올바른 수의 인수를 제공하지 않으면 프로그램은 디스플레이에 사용 메시지를 인쇄하고 종료해야 합니다.
(3) 입력 파일이 없는 경우 프로그램은 디스플레이에 오류 메시지를 올바르게 인쇄하고 종료해야 합니다.
(4) 사용자가 명령줄 인수로 프로그램을 올바르게 실행하면 프로그램은 "hw2_result.txt" 결과 파일을 생성해야 합니다.
(5) 출력 파일의 내용은 지정된 이진 파일을 실행하여 생성된 출력 파일과 같아야 합니다.
(6) input_long.txt 입력 파일의 실행 시간은 주어진 이진 파일 "hw2_example"과 유사해야 합니다. 프로그램의 실행 시간이 주어진 이진 파일에 비해 너무 길면 포인트가 차감됩니다.
5. 노트
- 여러분은 자신만의 코드를 작성해야만 합니다. 여러분은 다른 학생들과 아이디어를 논의할 수 있지만, 그들의 작품을 복사해서는 안됩니다. 비슷하게, 여러분은 인터넷에서 아이디어를 찾을 수 있지만, 인터넷에서 얻은 소스로부터 코드를 복사해서는 안됩니다.
- 우리 부서는 중복 코드를 탐지하는 소프트웨어를 사용합니다. 소프트웨어는 어셈블리 코드 레벨 탐지를 사용하기 때문에 변수 이름만 변경해도 소프트웨어를 우회하지 않습니다. 처음부터 코드를 작성해야 합니다. 그러면 문제가 없을 것입니다.
- 앞에서 설명했지만, 당신의 프로그램은 cspro 기계에서 컴파일하고 실행되어야 합니다. 당신의 작업물을 제출하기 전에 확인해야 합니다. TA는 당신의 코드를 다운로드하고 "make"를 실행하고 바이너리 파일이 올바른 출력을 생성하는지 확인하기 위해 실행할 것입니다.
